Backend Generator

Generate a complete Rust (Axum) backend API with 11 bd-component commands — no manual setup.

Getting Started CLI Reference Frontend Backend Templates Glass

Prerequisites: Run pinkboba bd-component scaffold first in an existing project. All commands must be run from the project root (where Pinkboba.toml exists).

Quick Start — Full Stack

cd my-project
pinkboba bd-component scaffold --with-db --with-env
pinkboba bd-component model User --fields name:String,email:String,age:i32
pinkboba bd-component controller User --fields name:String,email:String,age:i32
pinkboba bd-component auth
pinkboba bd-component middleware
pinkboba bd-component response
pinkboba bd-component migration create create_users_table
pinkboba bd-component migration run
pinkboba serve

scaffold

Generates a complete Rust crate at src/backhed/ with Axum 0.8, Tokio, serde, tower-http, and more.

pinkboba bd-component scaffold [--with-db] [--with-env]
FlagEffect
--with-dbAdds SQLite (rusqlite bundled), database.rs, migrations/ directory, DbPool type
--with-envAdds dotenvy, config.rs, .env, .env.example

Auto-generated: error.rs (AppError), lib.rs with AppState, main.rs with Axum server, models/mod.rs, controllers/mod.rs. Dependency serde_json = "1" always included.

The generated error.rs conditionally includes a Database(rusqlite::Error) variant — only when --with-db is used or rusqlite is in your Cargo.toml.

model

Generates a Rust struct with serde Serialize/Deserialize and auto-wires it into models/mod.rs.

pinkboba bd-component model <name> --fields name:type,...
pinkboba bd-component model User --fields name:String,email:String,age:i32
# → src/backhed/src/models/user.rs
# → Auto: pub mod user; in models/mod.rs

Supported types: String, i32, i64, f32, f64, bool. Unknown types default to String.

controller

Generates an Axum CRUD controller with 5 handlers (list, get_by_id, create, update, delete) and auto-wires routes.

pinkboba bd-component controller <name> --fields name:type,...
pinkboba bd-component controller User --fields name:String,email:String
# → src/backhed/src/controllers/user_controller.rs
# → Auto: pub mod user_controller; + .nest("/users", user_controller::router())
# in controllers/mod.rs

Smart auto-gen: If the model file doesn't exist yet, the controller command automatically creates a model stub and wires it. No pre-requisite ordering needed.

Each handler receives State<Arc<AppState>> and returns Result<..., AppError>. Unused parameters are automatically prefixed with _ to eliminate compiler warnings.

migration

SQL migration management — create timestamped .sql files, run pending migrations, or list existing ones.

pinkboba bd-component migration create <name>
pinkboba bd-component migration run
pinkboba bd-component migration list
# Create a migration
pinkboba bd-component migration create add_email_to_users
# → src/backhed/migrations/20250101000000_add_email_to_users.sql

# Edit the file, then run:
pinkboba bd-component migration run

Migration files are parsed for -- UP and -- DOWN sections. Only the -- UP section is executed. Already-applied migrations are tracked via a _migrations table and skipped on subsequent runs.

auth

Generates a complete JWT authentication system with real bcrypt password hashing and database queries.

pinkboba bd-component auth

Generated files:

  • src/backhed/src/auth/jwt.rs — create_token(), validate_token(), Claims struct with FromRequestParts extractor
  • src/backhed/src/auth/mod.rs — 3 routes: POST /auth/login, POST /auth/register, POST /auth/me

Features:

  • Login verifies bcrypt hash against users table
  • Register hashes password with bcrypt, inserts user
  • Me queries user by ID from JWT claims
  • Auto-creates users table if not exists
  • Auto-wires routes into controllers/mod.rs
  • Auto-generates error.rs if missing
  • Adds JWT_SECRET to src/backhed/.env

Dependencies added: jsonwebtoken, bcrypt, chrono, serde_json

Use the Claims extractor in any handler:

use crate::auth::jwt::Claims;

async fn me(claims: Claims) -> Result<Json<Value>, AppError> {
    // claims.sub = user_id, claims.email = email
}
Security: JWT expires after 24 hours. The secret key defaults to "change-me" when JWT_SECRET is not set. Always set a strong secret via pinkboba env set JWT_SECRET <your-secret> before production.

middleware

Generates CORS (permissive) and a request logger middleware, wired into the main router.

pinkboba bd-component middleware

Logs all requests: → GET /api/users on entry, ← 200 /api/users on response. Uses CorsLayer::permissive() from tower-http.

response

Generates ApiResponse<T> — a unified JSON response wrapper.

pinkboba bd-component response
// Success with data
ApiResponse::success(data)        // → { "success": true, "data": ... }

// Success with message
ApiResponse::success_msg("done")  // → { "success": true, "message": "done" }

// Error response (returns StatusCode + Json)
ApiResponse::error("not found")   // → (StatusCode, Json<{ "success": false, "message": "not found" }>)

The data field is automatically omitted when None.

validator

Generates validation helpers — simple functions that return Result<(), String>.

pinkboba bd-component validator
validator::required("")?;             // Err("Value is required")
validator::email("bad")?;             // Err("Invalid email format")
validator::min_length("ab", 3)?;      // Err("Must be at least 3 characters")
validator::max_length("abc", 2)?;     // Err("Must be at most 2 characters")

seeder

Generates a seeder::run() function for inserting sample data into the database.

pinkboba bd-component seeder
// Call in main.rs after database init:
seeder::run(&pool);

Inserts a sample user row by default — edit the generated file to add your own seed data.

pagination

Generates PaginationParams (query params extractor) and PaginatedResponse<T>.

pinkboba bd-component pagination
use crate::pagination::{PaginationParams, PaginatedResponse};

async fn list_users(pagination: PaginationParams) -> Json<PaginatedResponse<User>> {
    let page = pagination.page();              // default: 1
    let per_page = pagination.per_page();       // default: 10 (clamped 1-100)
    let offset = pagination.offset();           // (page - 1) * per_page
    let search = pagination.search_like();      // "%search_term%"

    PaginatedResponse::new(data, total, page, per_page)
    // → { "data": [...], "page": 1, "per_page": 10, "total": 42, "total_pages": 5 }
}

error

Generates AppError — the unified error type used by all controllers and auth handlers.

pinkboba bd-component error

This command is usually not needed — error.rs is auto-generated during scaffold and auth. Use this only if you need to regenerate it.

pub enum AppError {
    NotFound(String),
    Unauthorized(String),
    BadRequest(String),
    Internal(String),
    Database(rusqlite::Error),  // only if rusqlite is a dependency
}

// impl IntoResponse → returns JSON { success: false, message: "..." }
// impl From   → ? operator works with DB errors
// impl From        → ? operator works with status codes

Project Structure

After running the full setup, your backend looks like this:

src/backhed/
├── Cargo.toml
├── .env / .env.example
├── migrations/                  # .sql migration files
├── static/                      # Frontend assets (auto-copied)
└── src/
    ├── main.rs                  # Axum server entry point
    ├── lib.rs                   # AppState + module declarations
    ├── error.rs                 # AppError enum
    ├── config.rs                # Environment config parser
    ├── database.rs              # SQLite connection + migration runner
    ├── middleware.rs            # CORS + request logger
    ├── response.rs              # ApiResponse<T> wrapper
    ├── validator.rs             # Validation helpers
    ├── seeder.rs                # Seed data runner
    ├── pagination.rs            # PaginationParams + PaginatedResponse
    ├── auth/
    │   ├── mod.rs               # Auth routes (login/register/me)
    │   └── jwt.rs               # JWT create/validate + Claims extractor
    ├── models/
    │   ├── mod.rs               # Auto-wired model registry
    │   └── user.rs              # Model struct
    └── controllers/
        ├── mod.rs               # Auto-wired route registry
        └── user_controller.rs   # CRUD handlers
