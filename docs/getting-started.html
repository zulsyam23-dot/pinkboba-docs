Getting Started

Install Pinkboba and build your first project — frontend-only or full-stack with a Rust API.

Getting Started CLI Reference Frontend Backend Templates Glass Changelog

1. Install Pinkboba

If you have Rust installed, the easiest way is via Cargo:

cargo install pinkboba

Otherwise, download the binary for your platform from GitHub Releases and add it to your PATH.

Verify the installation:

pinkboba --version

2. Frontend-Only Project

Create a project in seconds — no build tools, no configuration.

Create a project

pinkboba new my-app

Creates my-app/ with index.html, style.css, script.js, and injection markers.

Add components

cd my-app
pinkboba fb-component navbar --style horizontal
pinkboba fb-component hero
pinkboba fb-component card
pinkboba fb-component theme --mode dark

Each component auto-injects its CSS, JS, and HTML into your project.

Preview

pinkboba serve

Open http://localhost:8080 in your browser.

Build for production

pinkboba build

Output goes to dist/ — ready to deploy anywhere.

3. Full-Stack Project (Rust + Axum)

Add a Rust backend with SQLite, JWT auth, and more — all from the CLI.

Create a project (or use existing)

pinkboba new my-api-app
cd my-api-app

Scaffold the backend

pinkboba bd-component scaffold --with-db --with-env

Generates a complete Rust crate at src/backhed/ with Axum 0.8, SQLite, env config, and AppError.

Initialize environment

pinkboba env init

Creates .env, .env.example, and updates .gitignore.

Generate model + controller

pinkboba bd-component model User --fields name:String,email:String,age:i32
pinkboba bd-component controller User --fields name:String,email:String

Model auto-wires to models/mod.rs. Controller auto-wires to controllers/mod.rs with full CRUD routes. If the model doesn't exist yet, the controller command creates it automatically.

Add authentication

pinkboba bd-component auth

Generates JWT auth with bcrypt password hashing. Auto-adds POST /auth/login, POST /auth/register, POST /auth/me routes and a users table. Sets JWT_SECRET in your .env.

Add extras (optional)

pinkboba bd-component middleware   # CORS + request logger
pinkboba bd-component response     # ApiResponse<T> wrapper
pinkboba bd-component validator    # Validation helpers
pinkboba bd-component pagination   # Pagination support

Run migrations

pinkboba bd-component migration create create_users_table
# Edit migrations/*.sql, then:
pinkboba bd-component migration run

Build and serve

pinkboba serve

Builds the Rust backend (cargo build --release), copies the frontend to src/backhed/static/, and spawns the server on http://localhost:3000. The API routes are served under /api/....

4. Database Admin Studio

After running migrations, explore your database with the built-in admin studio:

Web admin UI

pinkboba admin

Opens a web-based SQLite admin at http://localhost:9090. Features: table browser, CRUD, SQL editor, dark/light theme.

CLI database inspection

pinkboba admin tables
pinkboba admin table users
pinkboba admin query "SELECT * FROM users"
