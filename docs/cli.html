CommandDescription

helpPrint help information

pinkboba new

Scaffolds a new project from a built-in or external template.

pinkboba new [OPTIONS] <NAME>

Arguments:
  <NAME>  Project name (creates a directory with this name)

Options:
  -t, --template <TEMPLATE>  Template to use [default: default]
  -h, --help                 Print help

Available Templates

TemplateDescription
defaultStandard starter template with basic structure
minimalMinimal template with only essential files
professionalProfessional layout with hero, features, and gradient styling
showcaseMulti-page showcase with routing, counters, and 25+ pre-built components
glassGlassmorphism template with 27 glass components (downloaded from GitHub)

Examples

# Create with default template
pinkboba new my-app

# Create with professional template
pinkboba new my-app --template professional

# Create with glass template (downloads from GitHub)
pinkboba new my-app --template glass

pinkboba add

Adds a component or external package to the current project. Components are downloaded and injected into your project files.

pinkboba add <PACKAGE>

Arguments:
  <PACKAGE>  Component name (e.g. navbar) or package name (e.g. glass)

Options:
  -v, --version <VERSION>  Package version (for external packages)
  -h, --help               Print help

External Packages

PackageDescription
glass27 glassmorphism components with pg- prefix. Download from GitHub at runtime.

Examples

# Add a built-in component
pinkboba add navbar
pinkboba add card
pinkboba add theme

# Add an external package
pinkboba add glass

pinkboba remove

Removes a component or dependency from the current project.

pinkboba remove <SUBCOMMAND>

Subcommands:
  component     Remove a component from the project
  dependency    Remove a project dependency
  help          Print help

Examples

# Remove a component
pinkboba remove component navbar

# Remove a dependency
pinkboba remove dependency glass

pinkboba component

Generates a single component's CSS, JS, and HTML output without adding it to a project. Useful for previewing or extracting component code.

pinkboba component <COMPONENT>

Available Components:
  accordion    alert       avatar     badge      breadcrumb
  button       card        dropdown   file-upload footer
  form-login   form-signup hero       input      loading
  modal        navbar      oauth      otp        pagination
  progress-bar table       tabs       theme      toast
  toggle       tooltip

Examples

# Preview a button component
pinkboba component button

pinkboba build

Builds the project for production. Creates a dist/ directory with optimized output.

pinkboba build

pinkboba serve

Starts a local development server on http://localhost:8080 with automatic reload on file changes.

pinkboba serve

Environment Variables

VariableDescription
PINKBOBA_PACKAGES_PATH Local path to external packages directory. When set, Pinkboba loads packages from this path instead of GitHub. Useful for offline development or testing custom packages.

Example: 
$env:PINKBOBA_PACKAGES_PATH = "C:\Users\You\pinkboba-components"
