pinkboba new

Scaffolds a new project from a built-in or external template.

pinkboba new [OPTIONS] <NAME>

Arguments:
  <NAME>  Project name

Options:
  -t, --template <TEMPLATE>  Template [default: default]
  -h, --help                 Print help

Available Templates

TemplateDescription
defaultClean starter with injection markers
minimalMinimal boilerplate
professionalLanding page with hero, gradient styling
showcaseDemo with 25+ pre-built components, routing, theme toggle
glassGlassmorphism UI (downloaded from GitHub)
 
pinkboba new my-app
pinkboba new my-app --template showcase
pinkboba new my-app --template glass

pinkboba admin

Built-in SQLite database admin studio. Bisa diakses via browser (web UI) atau langsung dari terminal.

pinkboba admin [OPTIONS]
pinkboba admin <SUBCOMMAND> [OPTIONS]

Options:
  --db <PATH>  Path to SQLite database file (auto-detect .db files in src/backhed/)

Subcommands

SubcommandDescription
serveStart web admin studio on http://localhost:9090. Web UI: dark/light theme, CRUD rows, search, pagination, SQL editor, table manager (create/drop), server monitoring.
tablesList all tables in the database (CLI)
table <name>Show schema + preview 20 rows for a table
query "<SQL>"Run arbitrary SQL and display results
 
# Web admin studio
pinkboba admin
pinkboba admin serve                   # same as above
pinkboba admin serve --port 8080       # custom port

# CLI database inspection
pinkboba admin tables                  # list all tables
pinkboba admin table users             # schema + preview
pinkboba admin table users --db my.db  # specify db path

# CLI SQL runner
pinkboba admin query "SELECT * FROM users"
pinkboba admin query "SELECT name, email FROM users WHERE id = 1"

pinkboba add

Adds a package from the registry to the current project.

pinkboba add <PACKAGE> [--version <VERSION>]

Arguments:
  <PACKAGE>  Package name (e.g. glass)

Options:
  -v, --version <VERSION>  Package version
  -h, --help               Print help

Currently available packages: glass (27 glassmorphism components with pg- prefix, downloaded from GitHub at runtime).

pinkboba add glass

To use packages offline, clone pinkboba-components and set the PINKBOBA_PACKAGES_PATH environment variable to the local path.

pinkboba remove

Removes a component or dependency from the current project.

pinkboba remove <SUBCOMMAND>

Subcommands:
  component     Remove a frontend component from index.html
  dependency    Remove a dependency from Pinkboba.toml
pinkboba remove component navbar
pinkboba remove dependency glass

pinkboba fb-component

Generates a frontend UI component into src/components/ and auto-injects it into index.html.

pinkboba fb-component <SUBCOMMAND> [OPTIONS]
SubcommandOptions
button--variant primary / secondary / outline / danger / ghost
navbar--style horizontal / hamburger
theme--mode light / dark
form-login, form-signup, card, modal, alert, tabs, accordion, breadcrumb, dropdown, pagination, toggle, progress-bar, footer, loading, tooltip, hero, otp, oauth, table, toast, badge, avatar, file-upload—
 
pinkboba fb-component button --variant danger
pinkboba fb-component navbar --style hamburger
pinkboba fb-component card
pinkboba fb-component theme --mode dark

Each component generates a .pinkboba file in src/components/ and auto-injects its CSS <link>, JS <script>, and HTML tag at the corresponding injection markers.

pinkboba bd-component

Generates backend Rust (Axum) components. See the Backend Generator page for full documentation.

pinkboba bd-component <SUBCOMMAND> [OPTIONS]

Subcommands:
  scaffold    Generate base Rust Axum crate
  model       Generate serde struct, auto-wire
  controller  Generate CRUD handler, auto-wire
  migration   Create / run / list SQL migrations
  auth        Generate JWT auth (bcrypt login/register/me)
  middleware  Generate CORS + request logger
  response    Generate ApiResponse<T> wrapper
  validator   Generate validation helpers
  seeder      Generate seed data runner
  pagination  Generate pagination params + response
  error       Generate unified AppError enum
pinkboba bd-component scaffold --with-db --with-env
pinkboba bd-component model User --fields name:String,email:String
pinkboba bd-component controller User --fields name:String,email:String
pinkboba bd-component auth
pinkboba bd-component middleware

pinkboba env

Manage environment variables across root .env and src/backhed/.env.

pinkboba env <SUBCOMMAND>

Subcommands:
  init       Create .env, .env.example, update .gitignore
  set        Set an env var in all .env files
  get        Get an env var value
  list       List all env vars
  generate   Regenerate config.rs from .env values
pinkboba env init
pinkboba env set JWT_SECRET my-secret-key
pinkboba env get PORT
pinkboba env list
pinkboba env generate

env init also generates config.rs if scaffold exists, automatically adds .env to .gitignore, and creates .env.example files.

pinkboba build

Builds the project for production. Output goes to dist/.

pinkboba build
  1. Parse all .pinkboba files in src/, split into HTML/CSS/JS
  2. Copy non-.pinkboba files to dist/ (excluding backhed/)
  3. Compile Rust backend with cargo build --release
  4. Copy backend binary to dist/backhed(.exe)
  5. Strip injection markers from HTML

pinkboba serve

Starts the development server — builds, spawns backend (if available), and serves frontend.

pinkboba serve

Frontend only: Serves dist/ (or src/) via tiny_http on port 8080.

Full stack: Copies frontend to src/backhed/static/, spawns the backend binary, and serves everything from the Axum server (default port 3000).

Press Ctrl+C to stop.

Environment Variables

VariableDescription
PINKBOBA_PACKAGES_PATH Local path to external packages directory. When set, Pinkboba loads packages from this path instead of fetching from GitHub. Useful for offline development or custom packages. 
$env:PINKBOBA_PACKAGES_PATH = "C:\path\to\pinkboba-components"
PORT Frontend server port (default: 8080)
BACKHED_PORT Backend API server port (default: 3000)
JWT_SECRET Secret key for JWT token signing (set via pinkboba env set JWT_SECRET ...)
HOST Backend bind address (default: 0.0.0.0)
DATABASE_URL SQLite database file path (default: backhed.db)
APP_ENV Application environment (default: development)
