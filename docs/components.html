How It Works

Each component is defined as a .pinkboba file containing HTML template, CSS, and JavaScript. When you run pinkboba fb-component <name>:

  1. The component is extracted into src/components/ as a .pinkboba file
  2. CSS is injected as <link> at the <!--INJECT_CSS--> marker
  3. JavaScript is injected as <script> at the <!--INJECT_JS--> marker
  4. HTML tag is inserted at the <!--INJECT_HTML--> marker

Using Components

# Add a button with variant
pinkboba fb-component button --variant danger

# Add a navbar with style
pinkboba fb-component navbar --style hamburger

# Add a theme toggle (--mode light/dark)
pinkboba fb-component theme --mode dark

# Add components without options
pinkboba fb-component card
pinkboba fb-component hero
pinkboba fb-component modal
pinkboba fb-component form-login

Removing Components

pinkboba remove component navbar

This removes the component's CSS reference, JS reference, and HTML tag from index.html, then deletes the .pinkboba file from src/components/.

The .pinkboba Format

A .pinkboba file contains all three parts in one file:

<pb-button>Click Me</pb-button>

<style>
  pb-button { padding: 0.5rem 1rem; background: var(--pb-primary); }
</style>

<script>
  class PBButton extends HTMLElement { ... }
  customElements.define('pb-button', PBButton);
</script>

During pinkboba build, each .pinkboba file is split into separate .html, .css, and .js files in the dist/ output.

External Components (pg-*)

The glass package provides 27 additional components with the pg-* prefix, featuring glassmorphism styling with backdrop-filter: blur() and gradient accents. These are downloaded from GitHub at runtime.

pinkboba add glass
